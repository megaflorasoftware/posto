Posto is an editor for static, Markdown-based websites. It edits the files in a site's own
repository: non-programmers can update a site that was built for them, and developers get a
faster way to make frequent content changes. It runs as a desktop app (with a live preview) and
a mobile app.

:::caution[Pre-alpha software]
Posto is pre-alpha. Expect bugs and incomplete features, and use it at your own risk. The macOS
build is currently unsigned, so after installing you'll need to clear the quarantine flag:

```sh
xattr -d com.apple.quarantine /Applications/Posto.app/
```

:::

## What sites work with Posto?

Posto works on a Git repository of content files. How much it can do depends on how the site
describes its content:

- **Astro sites** get the deepest support. Posto generates typed editing forms from the site's
  [content collections](https://docs.astro.build/en/guides/content-collections/), inserts the
  site's own components into MDX, and manages image collections as
  [media libraries](/frameworks/astro/media-libraries/). See
  [Getting started with Astro](/frameworks/astro/getting-started/).
- **Any static site** — Hugo, Jekyll, Eleventy, or anything else that stores content as files —
  works by describing its content in a [Pages CMS](https://pagescms.org) `.pages.yml` file. See
  [Getting started with Pages CMS](/frameworks/pages-cms/getting-started/).
- **Everything else** still opens: Markdown files get the rich-text body editor and an untyped
  frontmatter form, and other files can be edited as raw text. You just don't get the typed
  forms.

These aren't alternatives. An Astro site can also carry a `.pages.yml` to add what Astro's
schemas can't express — field labels, media directories, dropdown option labels. When both are
present, `.pages.yml` takes precedence and Astro collections fill the gaps.

## Desktop and mobile

- **Desktop** (macOS, Windows, Linux) is the full experience: it sets up the site's local
  environment, runs the dev server, and shows a live preview beside the editor. It works on a
  repository already on your computer — clone it with Git first; Posto doesn't clone for you
  yet.
- **Mobile** is for editing on the go: sign in to GitHub and clone a site with one tap. There is
  no preview or dev server, so typed forms rely on schema files committed to the repository (the
  Astro guide [covers this](/frameworks/astro/getting-started/#recommended-gitignore)). Mobile
  builds aren't distributed yet — this page will link to them when they are.
