Posto edits static, Markdown-based websites directly in their Git repositories. It gives non-programmers structured controls for routine updates and gives developers a faster path through frequent content changes. The desktop app includes a live preview; the mobile app focuses on editing and publishing from GitHub.

:::caution[Pre-alpha software] Posto is pre-alpha. Expect bugs and incomplete features, and use it at your own risk. :::

## What sites work with Posto?

Posto can open any Git repository that contains editable content files. The site's schema determines how much structure Posto can add to the editing experience:

- **Astro sites** get the deepest support. Posto generates typed editing forms from the site's [content collections](https://docs.astro.build/en/guides/content-collections/), inserts the site's own components into MDX, and manages image collections as [media libraries](/frameworks/astro/media-libraries/). See [Getting started with Astro](/frameworks/astro/getting-started/).
- **Other static-site frameworks** — including Hugo, Jekyll, and Eleventy — can get typed forms from a [Pages CMS](https://pagescms.org) `.pages.yml` file. See [Getting started with Pages CMS](/frameworks/pages-cms/getting-started/).
- **Repositories without a supported schema** still open. Markdown files get a rich-text body editor and controls inferred from their frontmatter; other file types open as raw text.

Astro and Pages CMS configuration can work together. An Astro site can use `.pages.yml` for editor details that Zod schemas do not express, such as field labels, media directories, and labels for dropdown options. Where both describe the same content, `.pages.yml` takes precedence and Astro fills the gaps.

If a repository contains several sites, Posto asks which project directory to open. Editing, preview setup, and publishing then stay scoped to that project.

## Desktop and mobile

- **Desktop** (macOS, Windows, Linux) sets up the site's local environment, runs the development server, and shows a live preview beside the editor. Open a repository already on your computer; if it exists only on GitHub, clone it with Git or a Git client first.
- **Mobile** lets you sign in to GitHub and clone a repository in the app. It does not run a development server or show a preview, so typed Astro forms rely on schema files committed to the repository. The Astro guide [explains what to commit](/frameworks/astro/getting-started/#recommended-gitignore). Mobile builds are not distributed yet.
